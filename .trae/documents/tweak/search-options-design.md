# 搜索增强设计 - 下拉菜单方案

## 概述

在搜索栏的"搜索"按钮左侧添加一个"搜索选项 ▼"下拉菜单，提供两个开关选项：
1. **区分大小写**（默认关闭）
2. **搜索范围**：默认搜索全部（而非仅已加载项）

## 当前状态

搜索栏结构：`[🔄] [搜索框] [搜索] [+]`

- 普通文字：客户端过滤（`filteredKeys` computed），始终不区分大小写
- 含通配符（`* ? [`）：走后端 Redis KEYS 命令

## 设计方案

### 搜索栏新结构

```
[🔄] [搜索框] [搜索选项 ▼] [搜索] [+]
```

在搜索框和"搜索"按钮之间插入一个下拉菜单，风格与菜单栏的"排序 ▼"一致（`el-dropdown-link`）。

### 下拉菜单内容

```
┌─────────────────────────┐
│ ✓ 不区分大小写           │  ← 默认勾选，点击切换
│ ✓ 搜索全部               │  ← 默认勾选，点击切换为"仅已加载"
└─────────────────────────┘
```

使用 `el-dropdown-menu` + `el-dropdown-item`，每个选项前用图标表示勾选状态（✓/空），文字直接说明含义。

### 交互逻辑

#### 选项1：大小写敏感

| 状态 | 行为 |
|------|------|
| 不区分大小写（默认） | `filteredKeys` 用 `toLowerCase()` 匹配 |
| 区分大小写 | `filteredKeys` 用 `includes()` 直接匹配 |

#### 选项2：搜索范围

| 状态 | 行为 |
|------|------|
| 搜索全部（默认） | 先确保加载全部 keys，再过滤 |
| 仅已加载项 | 仅在当前已加载的 keys 中过滤 |

#### 搜索全部的实现

默认搜索全部时：
- 如果 keys 已全部加载（`keysCursor === 0`），直接过滤
- 如果未全部加载，自动调用 `handleLoadAll()` 加载全部后再过滤
- 加载过程显示进度条（复用现有"加载所有"逻辑）

### Placeholder 联动

搜索框的 placeholder 根据当前选项动态变化：
- 默认（搜索全部 + 不区分大小写）：`搜索全部 key`
- 区分大小写：`搜索全部 key（区分大小写）`
- 仅已加载项：`搜索已加载 key`
- 仅已加载 + 区分大小写：`搜索已加载 key（区分大小写）`

### 视觉样式

- "搜索选项 ▼"使用与"排序"相同的 `el-dropdown-link` 样式
- 下拉菜单项使用 `✓` 图标表示选中状态
- 当前有活跃选项时，"搜索选项"文字加粗提示

## 修改文件

### `src/frontend/views/MainView.vue`

1. **模板**：在搜索框和搜索按钮之间添加 `el-dropdown`
2. **状态**：添加 `caseSensitive` 和 `searchAll` 两个 ref
3. **filteredKeys computed**：根据 `caseSensitive` 切换匹配方式
4. **searchKeys 方法**：根据 `searchAll` 决定是否先加载全部
5. **placeholder**：computed 动态生成
6. **CSS**：复用现有 `el-dropdown-link` 样式

## 验证

1. 默认状态搜索行为与当前一致（不区分大小写、仅已加载项）
2. 开启"区分大小写"后，`User` 和 `user` 搜索结果不同
3. 开启"搜索全部"后，自动加载全部 keys 并搜索
4. 两者同时开启时正常工作
5. placeholder 随选项变化

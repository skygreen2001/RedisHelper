# 多选删除 + 废键箱功能实现计划

## 摘要

为 Redis 小助手（Tauri 2.0 + Vue 3 + Element Plus + Rust）添加**多选删除**和**废键箱**功能。删除的键不再直接从 Redis 中永久移除，而是先保存到本地废键箱（`~/.redis-helper/trash.json`），保留 7 天后自动清除。用户可在废键箱中查看、恢复或永久删除这些键。

---

## 当前状态分析

| 项目 | 现状 |
|------|------|
| 键列表组件 | `el-tree`，仅支持单击选中，无多选能力 |
| 删除操作 | `deleteKey()` 直接调用后端 `DEL` 命令，**无二次确认** |
| 数据存储 | 服务器配置存 `~/.redis-helper/config.json`，无废键箱存储 |
| 后端架构 | Tauri Command 模式，`commands/` + `redis/` + `storage/` 三层 |
| 前端架构 | Pinia Store + MainView.vue 单文件（约 1174 行） |

---

## 设计决策

| 决策点 | 选择 | 理由 |
|--------|------|------|
| 废键箱存储 | 本地 JSON 文件 `~/.redis-helper/trash.json` | 复用现有 ConfigManager 模式，无需引入新依赖 |
| 恢复目标 | 恢复到原 Redis 服务器和 DB | 保存删除时的完整连接信息（host/port/password/db） |
| 多选交互 | 工具栏"多选"切换按钮 | 不干扰日常单选操作，按需开启 |
| 废键箱 UI 入口 | 菜单栏按钮，右侧内容区切换 | 与现有 UI 风格一致，不新增页面 |
| 删除确认 | ElMessageBox 确认弹窗 | 与项目中删除服务器/数据库的交互一致 |
| 过期清理 | 每次打开废键箱时自动清理 + 应用启动时清理 | 确保用户始终看到最新状态 |

## 实现方案

### 一、后端 — 数据模型（新建 `storage/trash.rs`）

```
TrashItem {
    id: String,              // UUID
    key: String,             // Redis 键名
    value: String,           // 键值（JSON 序列化）
    key_type: String,        // string | list | set | zset | hash
    connection: TrashConnection,  // 来源连接信息
    deleted_at: String,      // 删除时间（ISO 8601）
    expires_at: String,      // 过期时间（deleted_at + 7天）
}

TrashConnection {
    host: String,
    port: u16,
    password: Option<String>,
    db: u8,
}

TrashData {
    items: Vec<TrashItem>,
}

TrashManager {
    data: TrashData,
    trash_path: String,
    // 方法: new(), save(), add_item(), remove_item(), get_items(), get_item(), cleanup_expired()
}
```

**设计要点**：
- 完全复用 `ConfigManager`（config.rs）的 JSON 文件读写模式
- `cleanup_expired()` 对比 `expires_at` 与当前时间，删除过期项
- `TrashConnection` 保存完整连接信息，确保恢复时精确还原

### 二、后端 — Tauri Commands（新建 `commands/trash.rs`）

| Command | 功能 | 参数 | 返回值 |
|---------|------|------|--------|
| `move_to_trash` | 单个键移入废键箱 | `{host, port, password, db, key}` | `bool` |
| `batch_move_to_trash` | 批量移入废键箱 | `{host, port, password, db, keys[]}` | `u32`（成功数量） |
| `get_trash_items` | 获取废键箱列表（自动清理过期项） | 无 | `Vec<TrashItemResponse>` |
| `restore_from_trash` | 恢复单个键 | `{trash_id}` | `bool` |
| `batch_restore_from_trash` | 批量恢复 | `{trash_ids[]}` | `u32` |
| `permanent_delete_trash` | 永久删除 | `{trash_ids[]}` | `u32` |
| `clear_expired_trash` | 手动清理过期项 | 无 | `u32` |

**`move_to_trash` 核心流程**：
1. 连接 Redis → 获取键值和类型
2. 构造 TrashItem（UUID + 时间戳）
3. 从 Redis 删除该键
4. 写入 trash.json

**`restore_from_trash` 核心流程**：
1. 从 trash.json 查找 TrashItem
2. 使用保存的 connection 连接原 Redis
3. 调用 `set_key_value()` 恢复
4. 从 trash.json 移除

### 五、前端 — UI 变更（修改 `MainView.vue`）

#### 5.1 菜单栏新增废键箱入口
- 在 DB 下拉菜单后新增"废键箱"按钮
- 带数量角标（`el-badge`），激活时高亮

- `v-if="!isTrashView"` → 现有键列表 + 值展示
- `v-else` → 废键箱视图（`el-table`）

#### 5.5 废键箱视图
- 工具栏：刷新、恢复选中、永久删除选中、清理过期项

#### 5.6 删除确认弹窗
- 单个删除：`ElMessageBox.confirm` → "确定要删除该键吗？删除后将移入废键箱，7天后自动清除。"
- 批量删除：`ElMessageBox.confirm` → "确定要将选中的 N 个键移入废键箱吗？7天后自动清除。"
- 成功提示：`"已移入废键箱，7天后自动清除"`

#### 5.7 新增 CSS 样式
- 多选按钮、批量删除按钮、废键箱按钮、废键箱视图、类型标签等样式

## 注意事项

1. **el-tree show-checkbox 动态切换**：Element Plus 不支持运行时动态切换 `show-checkbox`，采用 `v-if` 渲染两个 el-tree 实例解决
2. **批量删除性能**：在 `RedisConnection` 中新增 `delete_keys(keys: &[&str])` 方法，使用 `DEL key1 key2 ...` 一次性删除
3. **密码存储安全**：`TrashConnection` 中保存了 Redis 密码（本地文件，风险可控）
4. **恢复时键名冲突**：如果原键名已被重新使用，恢复操作会覆盖（与现有 set_key_value 行为一致）
5. **过期清理时机**：`get_trash_items` 每次调用自动清理 + 应用启动时清理

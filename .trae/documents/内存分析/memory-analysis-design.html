✨ 功能概述

📊

实时内存监控

显示 Redis 服务器的实时内存使用情况，包括已用内存、峰值内存、内存碎片率等关键指标。

📈

键类型分布分析

可视化展示 String、Hash、List、Set、ZSet 等不同类型键的数量和内存占用占比。

🏆

大键排行榜

自动扫描并排序内存占用最大的前 N 个键，帮助快速定位可能需要优化的对象。

💡

优化建议

基于内存使用情况，智能提供键类型转换、过期时间设置、数据压缩等优化建议。

方式一：独立菜单项

✓

方式二："更多"下拉菜单（推荐）

将内存分析功能放在"更多"下拉菜单中，保持菜单简洁，与导入/导出/日志等功能分组。当前已采用此方案。

3

方式三：对话框模式

作为模式对话框展示，适合需要聚焦分析的临时性查看，不影响主界面布局。

🔧 技术实现提示

后端命令

// Redis INFO memory 命令返回的关键指标 used_memory: 1342177280 // 已用内存（字节） used_memory_human: "1.25G" // 人类可读格式 used_memory_peak: 1589648384 string used_memory_peak_human: "1.48G" mem_fragmentation_ratio: 1.42 // 内存碎片率 maxmemory: 2147483648 // 最大内存限制 // 使用 DEBUG OBJECT 或 SCAN + TYPE 统计键类型 SCAN 0 COUNT 1000 TYPE key_name

数据流设计

1. 前端请求 → Tauri Command (get_memory_info) 2. Rust 后端执行 → redis::cmd("INFO").arg("memory") 3. 解析响应 → 转换为结构化 JSON 4. 前端渲染 → 仪表盘组件 // 异步扫描大键（防止阻塞） SCAN 0 COUNT 100 // 分批扫描 DEBUG OBJECT ENCODING key // 获取对象内存占用

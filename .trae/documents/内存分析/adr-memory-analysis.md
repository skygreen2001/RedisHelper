# Redis 内存分析功能设计文档

## 1. 功能概述

内存分析功能用于帮助用户监控和分析 Redis 服务器的内存使用情况，定位大键和内存占用异常，提供优化建议。

## 2. 功能需求

### 2.1 核心功能

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 实时内存监控 | 显示已用内存、峰值内存、内存碎片率等指标 | 高 |
| 键类型分布 | 可视化展示各类型键的数量和内存占比 | 高 |
| 大键排行榜 | 展示内存占用最大的前 N 个键 | 高 |
| Key 内存详情列表 | 显示每个键的内存占用大小，支持排序 | 高 |
| 优化建议 | 基于内存使用情况提供优化建议 | 中 |

### 2.2 UI 集成

**入口位置**：「更多」下拉菜单

```
菜单结构：
[连接 ▼] [DB ▼] [更多 ▼] [多选] [排序 ▼]
                  │
                  ├── 导入
                  ├── 导出
                  ├── 清空
                  ├── 生成测试数据
                  ├── 内存分析  ← 新增
                  └── 日志
```

## 3. UI 设计

### 3.1 整体布局

内存分析功能以对话框模式展示，包含以下区域：

1. **顶部标题栏**：标题 + 统计信息（总数、总大小）+ 刷新按钮
2. **仪表盘区域**：6 个关键指标卡片
3. **图表区域**：饼图（键类型分布）+ 柱状图（内存占用）
4. **大键排行榜**：Top 10 大键列表
5. **Key 详情列表**：所有键的内存占用列表

### 3.2 仪表盘指标

| 指标 | 说明 | 单位 |
|------|------|------|
| 已用内存 | 当前内存使用量 | GB/MB |
| 峰值内存 | 历史最高内存使用量 | GB/MB |
| 内存碎片率 | 内存碎片比例 | 比值 |
| 键总数 | 当前数据库键数量 | 个 |
| 过期键占比 | 设置过期时间的键占比 | % |
| 大键数量 | 超过阈值的大键数量 | 个 |

## 4. 技术实现

### 4.1 后端 API

**命令名**：`get_memory_info`

**请求参数**：
```rust
struct MemoryInfoRequest {
    host: String,
    port: u16,
    password: Option<String>,
    db: u8,
}
```

**返回数据**：
```rust
struct MemoryInfoResponse {
    used_memory: u64,           // 已用内存（字节）
    used_memory_human: String,   // 人类可读格式
    used_memory_peak: u64,      // 峰值内存
    mem_fragmentation_ratio: f64, // 碎片率
    maxmemory: u64,             // 最大内存限制
    keys_count: usize,          // 键总数
    expired_keys_ratio: f64,    // 过期键占比
    large_keys_count: usize,    // 大键数量
    key_memory_list: Vec<KeyMemoryItem>,
    key_type_stats: Vec<KeyTypeStat>,
}

struct KeyMemoryItem {
    key: String,
    size: u64,           // 字节
    size_human: String,   // 人类可读格式
    key_type: String,
}

struct KeyTypeStat {
    key_type: String,
    count: usize,
    memory_bytes: u64,
    memory_percent: f64,
}
```

### 4.2 前端数据流

```
1. 用户点击「更多」→「内存分析」
2. 前端调用 Tauri Command: `get_memory_info`
3. Rust 后端执行:
   - `INFO memory` 获取内存统计
   - `SCAN` 分批扫描所有键
   - `DEBUG OBJECT` 或 `MEMORY USAGE` 获取每个键的内存占用
   - `TYPE` 获取键类型
4. 返回结构化数据给前端
5. 前端渲染仪表盘和列表
```

### 4.3 性能优化

- **异步分批扫描**：使用 SCAN 命令分批获取键，避免阻塞
- **内存估算**：对于大数据量，采用抽样估算而非全量扫描
- **缓存机制**：缓存扫描结果，定期刷新
